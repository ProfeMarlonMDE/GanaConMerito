# Tagging Schema

## Objetivo

Normalizar etiquetas para recuperación selectiva de contexto.

## Reglas

- usar tags cortos y semánticos
- evitar sinónimos redundantes
- incluir módulo, dominio y tipo de artefacto cuando aplique

## Familias sugeridas

- módulos: `core`, `auth`, `dashboard`, `reportes`, `data`, `jobs`
- dominios: `arquitectura`, `delivery`, `calidad`, `ops`, `security`
- artefactos: `adr`, `feature`, `backlog`, `runbook`, `deuda`, `issue`
