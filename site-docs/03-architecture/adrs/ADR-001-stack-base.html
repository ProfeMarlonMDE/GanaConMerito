ADR-001: Stack base de GanaConMerito

Estado

review

Fecha

2026-04-23

Contexto

El repositorio actual muestra una base funcional apoyada en Next.js, Supabase y Docker. El proyecto viene de una operación anterior y requiere una decisión formal mínima que congele el stack base vigente para evitar cambios estructurales sin trazabilidad.

Decisión

Se adopta como stack base actual de referencia:

  • Next.js para aplicación principal
  • Supabase para autenticación, persistencia y operaciones de datos asociadas
  • Docker para empaquetado y ejecución controlada
  • Git + Markdown + scripts locales para sistema documental operativo

    • Alternativas consideradas

    1. Mantener el stack sin ADR formal. Rechazada por falta de trazabilidad.

    2. Replantear stack completo antes de estabilizar operación. Rechazada por alto riesgo y poca evidencia.

    3. Formalizar stack actual y revisar cambios mayores vía ADR posterior. Aprobación recomendada.

    Consecuencias

  • Todo cambio de stack o de componente estructural relevante requerirá nuevo ADR.
  • La prioridad se mueve a consistencia, operación y reducción de deuda invisible.
  • La documentación debe alinearse con este stack base mientras no exista sustitución aprobada.

    • Impacto en módulos

  • `platform`: referencia principal de ejecución
  • `auth`: acoplado a Supabase hasta nueva decisión aprobada
  • `data`: migraciones y modelo deben mantenerse coherentes
